MISTO HR Assistant People Operation Guide

연차·근태·ERP 개인정보·증빙 제출 문의를 업무 처리 관점으로 정리하는 HR 가이드 챗봇입니다.

V2 · Process Driven

업무 유형

V2 답변 원칙

  • 1. CASE 판단
    질문을 연차·근태·개인정보·급여 등 실무 유형으로 먼저 분류합니다.
  • 2. 처리 순서
    담당자가 바로 안내할 수 있도록 신청 경로, 승인자, 제출물, 기한을 분리합니다.
  • 3. 예외 관리
    사전 승인 누락, 출퇴근 누락, 증빙 미제출 등 리스크 항목을 따로 표시합니다.
  • 4. 보안 방어
    주민등록번호, 계좌번호, 비밀번호 등은 채팅창 입력을 제한하고 ERP 보안 화면 입력만 안내합니다.

민감정보 입력 금지

이 화면에는 주민등록번호 전체, 계좌번호 전체, 비밀번호, 가족의 상세 개인정보를 입력하지 마세요. 필요한 경우 ERP·사내 승인된 보안 경로에서만 처리합니다.

자주 쓰는 업무 질문

표준 답변 구조

1문의 유형 판단연차·근태·ERP·급여·증빙 중 어디에 해당하는지 분류
2CASE 분기정상 신청, 사후 정정, 예외 승인, 민감정보 포함 여부 확인
3처리 프로세스신청 경로 → 승인자 → 제출물 → 마감기한 순서 안내
4주의사항개인정보·증빙·급여 영향·반려 가능 사유 고지

CASE 판단 예시

연차사전 신청 원칙. 잔여 연차, 팀 승인, 업무 공백 여부 확인.
반차오전/오후 기준을 명확히 하고 근무시간 산정 기준 확인.
근태출퇴근 누락·지각·조퇴·외근은 사유와 증빙을 남겨 정정.
ERP주소, 연락처, 계좌, 가족정보 등 변경 항목별 입력 경로 안내.
보안주민등록번호·계좌번호 등은 채팅창 수집 금지. ERP 보안 화면만 사용.
